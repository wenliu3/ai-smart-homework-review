# 首次登录强制修改密码功能 - 前端实现说明

## 功能概述

实现了前端配合后端的首次登录强制修改密码功能，确保批量导入的用户和管理员重置密码的用户在首次登录或密码重置后必须修改密码才能继续使用系统。

## 实现细节

### 1. 类型定义更新

#### `types/auth.ts`

更新了登录结果和用户信息的类型定义：

```typescript
export interface LoginResult {
  token: string;
  refreshToken: string;
  expiresIn: number;
  userId?: string;

    id: string;
    username: string;
    email: string;
    name: string;
    role: string;
    mustChangePassword: boolean;
  };
}

export interface UserInfo {
  // ... 其他字段

  firstLoginAt?: string; // 首次登录时间

```

### 2. 用户状态管理更新

#### `store/modules/user.ts`

- **新增 getter**: `mustChangePassword` - 检查用户是否需要强制修改密码
- **更新登录 action**: 保存登录响应中的强制修改密码信息
- **更新用户信息获取**: 确保保留强制修改密码状态

```typescript
// 新增getter
const getters = {
  // ... 其他getters
  mustChangePassword: (state) => state.userInfo?.mustChangePassword || false
}

// 更新登录action
async login({ commit }, { usernameOrEmail, password, rememberMe = false }) {
  const response = await login({ usernameOrEmail, password, rememberMe })

  commit('SET_USER_INFO', {
    token: response.token,
    refreshToken: response.refreshToken,
    tokenExpiresAt: Date.now() + response.expiresIn * 1000,
    mustChangePassword: response.mustChangePassword,
    isFirstLogin: response.isFirstLogin,
    ...(response.user && response.user)
  })

  return response
}
```

### 3. 强制修改密码组件

#### `views/ForceChangePassword.vue`

创建了专门的强制修改密码页面：

**主要功能**：

- 显示友好的提示信息（区分首次登录和管理员重置）

- 表单验证和错误处理
- 修改成功后自动清除强制修改标记
- 支持重定向到原始访问页面

### 5. 登录流程增强

#### `views/Login.vue`

在登录成功后检查是否需要强制修改密码：

```typescript
// 登录逻辑
const loginResponse = await store.dispatch("user/login", {
  usernameOrEmail: form.email,
  password: form.password,
  rememberMe: form.rememberMe,
});

ElMessage.success("登录成功");

// 检查是否需要强制修改密码
if (loginResponse.mustChangePassword) {
  // 跳转到强制修改密码页面，并保存原始跳转地址
  const redirectUrl = (route.query.redirect as string) || "/";
  router.push(
    `/force-change-password?redirect=${encodeURIComponent(redirectUrl)}`
  );
} else {
  // 正常跳转
  router.push((route.query.redirect as string) || "/");
}
```

### 6. 路由配置与守卫

#### `router/index.ts`

添加了强制修改密码页面的路由：

```typescript
{
  path: '/force-change-password',
  name: 'ForceChangePassword',
  component: () => import('../views/ForceChangePassword.vue'),
  meta: { requiresAuth: true, skipPasswordCheck: true }
}
```

#### `router/permission.ts`

添加了路由前置守卫检查：

```typescript
// 检查是否需要强制修改密码
const mustChangePassword = store.getters["user/mustChangePassword"];

// 如果需要强制修改密码且不是强制修改密码页面，则重定向
if (
  mustChangePassword &&
  to.path !== "/force-change-password" &&
  !to.meta?.skipPasswordCheck
) {
  next(`/force-change-password?redirect=${encodeURIComponent(to.fullPath)}`);
  NProgress.done();
  return;
}
```

## 用户体验流程

### 场景 1：批量导入用户首次登录

1. **用户使用初始密码登录**（`123456789`）
2. **系统检测到强制修改密码标记**
3. **自动跳转到强制修改密码页面**
4. **显示首次登录提示**："检测到您是首次登录，为了保护您的账户安全，请立即修改默认密码。"
5. **用户填写当前密码和新密码**
6. **系统验证密码强度**
7. **修改成功后跳转到原始访问页面**

### 场景 2：管理员重置密码后

1. **用户使用重置后的密码登录**
2. **系统检测到强制修改密码标记**
3. **自动跳转到强制修改密码页面**
4. **显示密码重置提示**："管理员已重置您的密码，为了保护您的账户安全，请立即修改密码。"
5. **用户完成密码修改流程**

### 场景 3：正常用户登录

1. **用户正常登录**
2. **系统检测无强制修改密码需求**
3. **直接跳转到目标页面**

## 配置说明

### 路由元信息

```typescript
meta: {
  requiresAuth: true,        // 需要登录
  skipPasswordCheck: true    // 跳过密码检查（用于强制修改密码页面本身）
}
```

### 白名单路由

强制修改密码检查不会拦截以下页面：

- `/force-change-password` - 强制修改密码页面
- 所有标记了 `skipPasswordCheck: true` 的路由

## 注意事项

1. **强制修改密码页面本身需要登录才能访问**，这确保了安全性
2. **修改密码后会立即清除前端状态**，无需重新登录
3. **支持重定向机制**，用户修改密码后可以继续访问原始页面
4. **路由守卫会在每次导航时检查**，确保强制修改密码的有效性

这个实现提供了完整的前端强制修改密码功能，与后端接口完美配合，确保用户账户安全。
